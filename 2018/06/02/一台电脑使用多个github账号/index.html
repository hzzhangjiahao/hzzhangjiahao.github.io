一台电脑使用多个github账号

Posted on 2018-06-02 | In 拾贝
Words count: 678

通常电脑都是自己一个人用，所以 github 、gitlab 等网站都使用同一个 ssh key 进行鉴权，这个 key 就代表你自己。但难免会有特殊情况，比如 GF 也用我的电脑往 github 提交代码，默认 key 只能被一个 github 账号绑定，此时就需要针对多账号进行多个 ssh key 的配置。

问题分析

我们知道 ssh key 是用来鉴权的，一个 key 代表一个人(角色)，在同一个 git server (例如github) 上只能绑定一个账号，如果两个人在同一个 git server 上提交自然就需要两个 ssh key，那么问题来了：

  1. 如何在一台电脑上区分这两个 key？
  2. 如何在提交时透明地选择正确的 key 进行鉴权？

调查线索

查看资料，其中 Host 可以理解为这套配置的别名，Hostname 是 git server 的域名，User 是账号名，IdentityFile 则指定了 key。此前没有手动维护过这个文件，这些配置应该是第一次 git clone 的时候自动生成的，github 使用 ssh clone 的 url 是这样的：

不难发现@后面的一段内容就对应 config 中的 Host。这样思路就理清了，在 config 中使用两条配置，指定不同的 ssh key，使用不同的 Host 进行区分，git repo 的 url 中的 host 与 Config 中的 Host 一一对应，这样对 repo 进行 push 的时候，就可以通过 url 找到配置项，确定 HostName 和 ssh key，透明地完成了选择匹配的账号进行提交的工作。

实施步骤

  1. 生成 ssh key: ssh-keygen -t rsa -C "your_email@example.com"

  2. 将 ssh 私钥增加到 ssh-agent: ssh-add ~/.ssh/github_zhangjiahao_rsa

  3. 复制 ssh 公钥到剪贴板: pbcopy < ~/.ssh/github_zhangjiahao_rsa.pub，绑定到 github 账号。
